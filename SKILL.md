# Document to Canvas Skill - 主题式知识提取

**Author:** [Ryan Chen](https://x.com/fidoyeah)

将各种文档转换为**主题驱动**的思维导图，聚焦核心思想、理论框架和可操作方法，而非简单罗列章节。

## 核心理念

### ❌ 传统做法（章节导向）
- 第一章：引言
- 第二章：概念A
- 第三章：概念B
- ...（读者迷失在结构里）

### ✅ 优化做法（主题导向）
- **核心问题**：本书解决什么痛点？
- **理论框架**：作者提出的核心模型/方法论
- **关键概念**：3-5个必须理解的核心概念
- **实践工具**：可立即应用的模板/清单/步骤
- **常见误区**：需要避免的陷阱
- **行动方案**：从认知到实践的桥梁

## 输出目录结构

将 `YOURPATH` 替换为你的 Obsidian 库中用于存放 Canvas 的目录（应包含 `书` 与 `文章` 两个子文件夹）。也可设置环境变量 `DOCUMENT_TO_CANVAS_OUTPUT_BASE` 指向该目录。

```
YOURPATH/
├── 书/                    # 书籍类文档输出目录
│   └── [书名]_主题框架.canvas
└── 文章/                  # 网络文章输出目录
    └── [文章标题]_核心洞见.canvas
```

## 内容提取方法论

### 第一步：识别核心命题
阅读时不断问：
- 如果全书只能记住一句话，是什么？
- 作者最想改变读者的什么认知？
- 这本书与同类书的区别在哪里？

### 第二步：寻找理论框架
寻找：
- 对比表格（如"求职者vs购物者"）
- 流程图/步骤图
- 数学公式或模型
- 自创术语（如"弱关系"、"职业安全"）

### 第三步：提取可操作方法
寻找：
- "步骤"、"流程"、"方法"
- 具体对话/邮件模板
- 检查清单
- 案例中的具体行动

### 第四步：识别反常识观点
寻找：
- "大多数人认为...但事实是..."
- "违背直觉的是..."
- 与主流观点相反的结论

## 质量检查清单

- [ ] 是否能用一句话概括核心命题？
- [ ] 是否识别了至少1个理论框架？
- [ ] 是否提取了3-5个关键概念？
- [ ] 是否提供了立即可用的工具？
- [ ] 是否指出了常见误区？
- [ ] 是否有具体的行动方案？
- [ ] 节点之间是否有逻辑关联（非简单罗列）？
- [ ] 是否适合快速浏览（标题清晰）？
- [ ] 是否便于实践应用（有 checklist）？

## 注意事项

1. **深度优于广度**：宁可深挖3个概念，不要罗列10个章节
2. **连接胜于堆砌**：展示概念间的因果关系
3. **行动胜于认知**：每个理论都配套实践方法
4. **反直觉优于常识**：突出颠覆性观点
5. **视觉化优于文字**：使用表格、清单、流程图
